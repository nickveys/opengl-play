File Descriptions

Each file is documented internally, here is an overview of them all:

Model.{h,cpp}
Base class for shapes that can be added to the screen
Cone.h, Cube.h, Sphere.h, Teapot.h, Torus.h
Concrete shapes
Matrix.{h,cpp}
Supplied Matrix class with some modification (documented)
Util.h
Utility class, contains function to write strings to display
enums.h
Holds some enums used in the system
main.cpp
Entry point

Extra Features

  • Bottom panel starts disabled as there is no object to manipulate
  • Many different shapes
  • Many different colors
  • Current shape can be spun continuously
  • Current shape can be set to "throb" (grow and shrink periodically)
  • The model axis can be optionally shown for the Current shape, All shapes or No shapes
  • OpenGL capability text is written to the screen instead of stdout

Known Problems

No real bugs, just some nuisances I didn't think were too worthy of a lot of extra effort.

  • The bottom panel is sometimes underneath the right panel, hiding the Scale spinners. Not sure how to fix this other than making the window larger.
  • The line spacing of the OpenGL capabilities text on the screen spaces out as the window is resized.
  • The strings written to screen could get truncated, I allocated 80 characters for them, which was fine for my machine.
  • The rotation sliders tend to jump to a random location after the "spin" animation has been running.
  • GLUI seems to use 100% CPU when idle, demo program also does it, plenty of people on the internet with the same problem.
